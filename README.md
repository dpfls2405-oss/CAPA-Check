# ⚡ CAPA Check
> 시디즈(평택) 포장라인 재고 적정성 & CAPA 점검 시스템

SCP 계획 + 출고량 실적 + 생산실적을 업로드하면  
**재고 과부족 판정 · 포장라인 CAPA 점검**을 자동으로 수행하는 Streamlit 앱입니다.

---

## 🚀 설치 및 실행

### 1. Python 환경 준비 (Python 3.9 이상)
```bash
# 가상환경 생성 (권장)
python -m venv venv
venv\Scripts\activate      # Windows
source venv/bin/activate   # Mac/Linux
```

### 2. 패키지 설치
```bash
pip install -r requirements.txt
```

### 3. 앱 실행
```bash
streamlit run app.py
```
브라우저에서 자동으로 http://localhost:8501 열림

## 📋 매월 사용법

### 1단계: 파일 준비
| 파일 | 설명 | 비고 |
|------|------|------|
| **SCP 계획 파일** | `_시디즈_SCP_YYMMDD_...xlsx` | 필수 |
| **출고량 실적 파일** | `그룹사_의자_출고량_*.csv` | 필수 |
| **생산실적 파일** | 라인별/품목별 생산실적 xlsx or csv | 선택 |

### 2단계: 앱 설정
- **기준월**: 분석 대상 월 (예: 202604)
- **전월 영업일**: 전달 영업일 수
- **당월 영업일**: 이번 달 영업일 수

### 3단계: 분석 실행
`데이터 업로드` 탭 → 파일 업로드 → `🚀 분석 실행` 클릭

### 4단계: 결과 확인
- **대시보드**: 전체 현황 한눈에
- **재고 적정성**: 라인별/품목별 비율 확인
- **CAPA 점검**: 일일 부하량 & 가동률
- **과소 품목**: 재고 부족 품목 + 자동 한 줄평
- **생산실적**: 목표 대비 생산 달성률

## 🔢 계산 기준

### 재고 적정성 비율
```
2주치 기준 = 전월 실적 ÷ 2
비율(%) = 당월 목표재고 ÷ 2주치 × 100
```

### 판정 기준
| 판정 | 비율 | 의미 |
|------|------|------|
| 🔴 과소 | < 80% | 재고 부족 위험 |
| 🟢 적정 | 80 ~ 120% | 정상 |
| 🟡 다소과다 | 120 ~ 200% | 과잉 검토 |
| 🟠 과다 | > 200% | 재고 조정 필요 |

### 일일 부하량
```
일일 부하량 = 월 목표재고 ÷ 영업일수
부하 변화율(%) = (당월 일일부하 - 전월 일일부하) ÷ 전월 일일부하 × 100
CAPA 가동률(%) = 일일 부하량 ÷ 라인 일일 CAPA × 100
```

## ⚙️ 커스터마이징

### 포장라인 매핑 수정
`데이터 업로드` → `포장라인 매핑 설정` 에서 수정  
또는 `utils/parser.py`의 `DEFAULT_LINE_MAP` 딕셔너리 직접 수정

### CAPA 설정 변경
`데이터 업로드` → `포장라인별 일일 CAPA`  
또는 `utils/parser.py`의 `DEFAULT_CAPA` 딕셔너리 수정

### 생산실적 파일 포맷
아래 컬럼이 포함되면 자동 인식 (유사 명칭도 OK):
- `포장라인` / `라인` / `LINE`
- `조합코드` / `단품코드` / `COMBO`  
- `생산수량` / `수량` / `실적`
- `일자` / `날짜` / `연월` (선택)
